What is a bug?

  • Historical Perspective
  • At the heart of a bug - invalid assumption
  • Result of not testing enough
  • Distinction: Bug, Error, Fault, Defect, Failure

Debugging

  • Debugging is like being a detective:
    • In a case where you’re the perpetrator as well :-D
  • Correcting bug is easy, difficult part is:
    • reproducing it
    • understanding it

Recipe for Debugging

  1. Reproduce the bug
  2. Start to look, stop making assumptions
  3. Divide and conquer - interval halving
  4. Change one thing at a time
  5. Fix the bug - correct the code
  6. Add a test to prevent the bug from raising from death unnoticed
  7. Share your experience
