Refactoring

A workshop by Martin Užák

2023-07-14

Refactoring?

  • Improve code without changing the functionality

Why?

  • 2/3 of software projects fail
  • 2/3 of development costs is spent in maintenance

Maintenance

  • What is maintenance?
    • bug-fixing
    • implementing new functionality
  • What prevents maintenance?

Technical Debt

  • What is it?
    • Nomen est omen
    • Insufficient attention paid to technical issues
  • How does it happen?
  • How do we resolve it?
    • by refactoring

Refactoring: when?

  • immediately
  • OR mark the mines in code / create an issue
    • 2 minute rule (time management)
  • periodically

Refactoring: how?

  • Understand the code you want to refactor
  • Verify your understanding (run it)
  • Create a test
  • Refactor
    • REPEAT (refactoring is an iterative process)

Goals of Refactoring

  • KISS, SRP: Complexity -> Simplicity
  • DRY: Repetitions (Copy&Paste) -> Abstractions
  • Naming, Structure -> Clarity

Code Smells & Corrections

  • Naming
  • Magic Constants
  • Copy & Paste vs. Functions (DRY)
  • Formatting
  • Dead Code
  • Useless Comments
  • KISS 0 > 1 > 2 > … too many
  • Main Function

Example

refactor.py -> refactored.py
